# Task 6

В рамках задания было настроено проксирование запросов через **Nginx**.

Запросы перенаправляются в Docker-контейнер с FastAPI-приложением.

Схема работы:

`localhost:8082 -> nginx:80 -> app:8081`

- контейнер `nginx_proxy` принимает запросы
- внутри контейнера Nginx слушает `80` порт
- далее запросы проксируются в контейнер `app`, работающий на порту `8081`

# Task 7

В рамках задания была реализована блокировка доступа по IP-адресу на уровне **Nginx**.

Проверка выполняется на стороне веб-сервера, без участия FastAPI-приложения.  
Если IP клиента попадает в список российских IP-префиксов, Nginx выполняет редирект на страницу-заглушку.  
Если IP не попадает в список, запрос проксируется в контейнер с приложением.

Схема работы:

`client -> nginx -> проверка IP -> /stub или app:8081`

Для локального тестирования использовался заголовок `X-Forwarded-For`, так как при обращении к `localhost` Nginx видит локальный адрес клиента, а не внешний публичный IP.

IP-префиксы были получены из базы **GeoLite2-Country.mmdb**
